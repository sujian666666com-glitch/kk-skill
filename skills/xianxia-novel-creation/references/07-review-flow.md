# 创作审核流程

---

## 一、章节审核

### 审核清单

**基础检查**
- [ ] 字数达标（3000-5000字/章）
- [ ] 无错别字、病句
- [ ] 标点符号正确
- [ ] 段落划分合理

**内容检查**
- [ ] 章节目标达成
- [ ] 起承转合完整
- [ ] 伏笔已埋设
- [ ] 悬念设置合理

**角色检查**
- [ ] 角色性格一致
- [ ] 对话符合人设
- [ ] 无角色崩人设

**战力检查**
- [ ] 境界战力自洽
- [ ] 战斗逻辑合理
- [ ] 无越级崩坏

**伏笔检查**
- [ ] 本章伏笔已埋
- [ ] 旧伏笔未遗漏
- [ ] 伏笔符合伏笔图

**衔接检查**
- [ ] 与上章衔接自然
- [ ] 为下章铺垫到位
- [ ] 无前后矛盾

---

### 章节质量评级

| 评级 | 标准 |
|-----|------|
| S | 情节精彩+文笔流畅+伏笔巧妙+有记忆点 |
| A | 情节完整+节奏合适+伏笔到位 |
| B | 情节通顺+基本达标 |
| C | 需要修改 |
| D | 需大幅重写 |

---

### 常见问题

| 问题 | 表现 | 解决方案 |
|-----|------|---------|
| 节奏拖沓 | 废话多、节奏慢 | 删减过渡、直接推进 |
| 节奏太快 | 信息量过大 | 增加过渡、放缓节奏 |
| 角色崩人设 | 行为不符合性格 | 回溯角色设定 |
| 战力崩坏 | 越级太容易/太难 | 重新规划实力对比 |
| 伏笔遗漏 | 之前埋的忘了回收 | 检查伏笔图 |
| 逻辑漏洞 | 前后矛盾 | 通读检查 |

## 二、卷审核

### 审核清单

**结构检查**
- [ ] 章节数量合理（20-30章/卷）
- [ ] 篇幅达标（10-20万字/卷）
- [ ] 起承转合完整
- [ ] 高潮低谷分布合理

**目标检查**
- [ ] 卷目标达成
- [ ] 主角成长明显
- [ ] 核心事件完成
- [ ] 情感变化到位

**伏笔检查**
- [ ] 本卷伏笔已埋设
- [ ] 上卷伏笔已回收
- [ ] 下卷伏笔已规划
- [ ] 伏笔逻辑自洽

**角色检查**
- [ ] 主角弧光完整
- [ ] 配角戏份合适
- [ ] 反派塑造到位
- [ ] 新角色登场自然

**高潮检查**
- [ ] 有明显高潮章节
- [ ] 高潮前铺垫到位
- [ ] 高潮后收束合理
- [ ] 情绪释放充分

**衔接检查**
- [ ] 与上卷衔接自然
- [ ] 为下卷铺垫到位
- [ ] 无战力断层
- [ ] 无设定矛盾

---

### 卷质量评级

| 评级 | 标准 |
|-----|------|
| S | 节奏完美+高潮迭起+情感充沛+伏笔精妙 |
| A | 结构完整+节奏合适+伏笔到位 |
| B | 基本达标+可读性强 |
| C | 需要修改 |
| D | 需大幅重写 |

---

### 卷审核问题

| 问题 | 表现 | 解决方案 |
|-----|------|---------|
| 节奏单调 | 全程紧张或全程平淡 | 增加节奏变化 |
| 高潮不足 | 平铺直叙无爆点 | 强化核心冲突 |
| 支线失衡 | 支线喧宾夺主 | 精简支线 |
| 成长不足 | 主角进步太慢 | 增加历练机会 |
| 伏笔混乱 | 伏笔太多/太乱 | 整理伏笔系统 |
| 角色退场 | 重要角色突然消失 | 补充交代 |

## 三、大纲审核

### 审核清单

**完整性检查**
- [ ] 主线完整无断点
- [ ] 辅线有起承转合
- [ ] 伏笔有埋有收
- [ ] 结局明确

**逻辑检查**
- [ ] 境界提升合理
- [ ] 冲突升级自然
- [ ] 角色动机合理
- [ ] 因果关系自洽

**节奏检查**
- [ ] 高潮分布合理
- [ ] 低谷有转折
- [ ] 张弛节奏合适
- [ ] 无拖沓阶段

**伏笔检查**
- [ ] 核心伏笔已规划
- [ ] 伏笔埋设时机合理
- [ ] 伏笔回收有安排
- [ ] 悬念设置有吸引力

**角色检查**
- [ ] 主角成长轨迹清晰
- [ ] 配角有各自弧光
- [ ] 反派有合理动机
- [ ] 关系网完整

## 四、审核流程

### 个人创作审核

```
创作完成
    ↓
章节自审 → 修改 → 通过
    ↓
卷内审核 → 调整 → 通过
    ↓
卷完结审核 → 整体修改 → 通过
    ↓
大纲对照审核 → 修订 → 完成
```

### 二审流程（可选）

```
作者提交
    ↓
一审（错别字/格式）→ 反馈修改
    ↓
二审（情节/逻辑）→ 反馈修改
    ↓
终审（整体质量）→ 通过/打回
    ↓
完成
```
