# compliance-gap-analysis

Assess an organization's security and privacy controls against a regulatory framework and produce a structured gap analysis report.

## What it does

This skill guides a compliance officer, security analyst, or auditor through a structured interview covering all control domains of a chosen framework. It produces:

- A **gap analysis matrix** mapping each requirement to Met / Partial / Gap status with severity ratings
- An **executive summary** with key metrics and overall compliance posture
- A **prioritized remediation roadmap** sorted by severity and effort
- An **assumptions and limitations** section flagging unverified areas

## Supported frameworks

| Framework | Scope |
|---|---|
| GDPR | EU personal data handling and privacy rights |
| SOC 2 Type I / II | SaaS vendor security posture |
| ISO 27001 | Enterprise information security management |
| HIPAA | US healthcare protected health information |
| PCI-DSS | Payment card data environments |
| NIST CSF | US federal / critical infrastructure cybersecurity |
| Other | Any framework the user names |

## When to use

- First-time readiness assessment before a formal audit
- Re-assessment after a previous audit cycle to track progress
- Scoping exercise before hiring an external auditor
- Board or executive reporting on compliance posture

## Safety and limitations

- Findings are based on self-reported controls and are not independently verified.
- This skill does not provide legal advice. Consult qualified legal counsel for regulatory interpretations.
- Sensitive details shared during the session (employee names, credentials, customer data) should be the minimum necessary to complete the analysis.
