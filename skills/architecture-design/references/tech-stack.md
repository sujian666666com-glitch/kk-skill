# Agent 技术选型矩阵

## 一、LLM 选型

| 模型 | 优势 | 劣势 | 适用场景 | 价格区间 |
|------|------|------|---------|---------|
| GPT-4o | 综合能力强，工具调用稳定 | 成本较高 | 复杂推理、多工具调用 | $$$ |
| GPT-4o-mini | 性价比高，速度快 | 复杂任务能力弱 | 简单分类、格式转换 | $ |
| Claude 3.5 Sonnet | 长文本处理优，代码能力强 | 工具调用不如 GPT | 代码生成、长文档分析 | $$ |
| Claude 3 Haiku | 速度快，成本低 | 能力有限 | 简单任务、高并发 | $ |
| 通义千问 | 中文能力强，国内合规 | 工具调用生态不成熟 | 国内业务、中文场景 | $$ |
| GLM-4 | 国内可用，中文好 | 综合能力弱于 GPT-4 | 国内业务 | $$ |

### 选型决策树
```
需要强推理能力？
├── 是 → GPT-4o / Claude 3.5
│   └── 预算有限？ → Claude 3.5 Sonnet
├── 否 → 简单分类/提取？
│   ├── 是 → GPT-4o-mini / Claude 3 Haiku
│   └── 否 → 中文为主？
│       ├── 是 → 通义千问 / GLM-4
│       └── 否 → GPT-4o-mini
```

## 二、编排框架

| 框架 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| LangGraph | 状态管理灵活，可视化好 | 学习曲线较陡 | 复杂状态机、多步骤任务 |
| CrewAI | 多 Agent 协作简单 | 定制性有限 | 多角色协作场景 |
| AutoGen | 微软支持，多 Agent 强 | 文档不够完善 | 研究性项目 |
| Dify | 低代码，部署快 | 深度定制难 | 快速原型、内部工具 |
| 自研 | 完全可控 | 开发成本高 | 核心产品、高定制需求 |

### 选型建议
- **快速验证**：Dify / CrewAI
- **生产级产品**：LangGraph / 自研
- **多 Agent 协作**：CrewAI / AutoGen
- **简单单 Agent**：直接调用 LLM API，无需框架

## 三、向量数据库

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| Pinecone | 托管服务，零运维 | 成本较高，数据出境 | 快速启动、国际业务 |
| Qdrant | 开源，性能好 | 需要自运维 | 自建部署、数据合规 |
| Milvus | 开源，分布式支持 | 部署复杂 | 大规模数据 |
| Chroma | 轻量，集成简单 | 功能有限 | 原型开发、小规模 |
| pgvector | 复用 PG，无额外组件 | 性能不如专用方案 | 已有 PG 的团队 |

### 选型建议
- **快速启动**：Pinecone / Chroma
- **数据合规（国内）**：Qdrant / Milvus
- **已有 PostgreSQL**：pgvector
- **大规模（百万+文档）**：Milvus

## 四、部署方案

| 方案 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| Docker + K8s | 标准化，可扩展 | 运维复杂 | 生产环境、多服务 |
| Serverless（Lambda/云函数） | 按需付费，零运维 | 冷启动、超时限制 | 低频调用、简单任务 |
| 裸机/VPS | 成本低，完全控制 | 运维负担重 | 开发测试、小团队 |
| PaaS（Railway/Render） | 部署简单 | 定制性有限 | 快速上线、MVP |

### 选型建议
- **MVP/原型**：PaaS / Serverless
- **生产环境**：Docker + K8s
- **长时运行任务**：避免 Serverless（超时限制）
- **高并发**：K8s + 自动扩缩容

## 五、监控与可观测性

| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| LLM 调用追踪 | LangSmith / LangFuse | 追踪每次 LLM 调用的输入输出和延迟 |
| 应用监控 | Prometheus + Grafana | 指标采集和可视化 |
| 日志 | ELK / Loki | 结构化日志存储和查询 |
| 告警 | PagerDuty / 自建 | 异常检测和通知 |

## 六、成本估算参考

### LLM 调用成本（每百万 token）
| 模型 | 输入 | 输出 |
|------|------|------|
| GPT-4o | $2.50 | $10.00 |
| GPT-4o-mini | $0.15 | $0.60 |
| Claude 3.5 Sonnet | $3.00 | $15.00 |
| Claude 3 Haiku | $0.25 | $1.25 |

### 优化策略
1. **分级调用**：简单任务用小模型，复杂任务用大模型
2. **缓存**：相似请求缓存结果
3. **批处理**：合并多个请求
4. **Prompt 压缩**：减少不必要的 token
