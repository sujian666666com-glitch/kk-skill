# 写作复盘模板

这个文件用于记录写作后期修改中暴露的问题。它不是风格素材，也不是写作素材；它是下一次写作前必须读取的避错清单。

建议把实际复盘文件命名为 `写作复盘.md`，放在本次写作输出目录或用户长期维护的写作目录中。不要把包含私人反馈、客户要求、未公开文章内容的 `写作复盘.md` 上传到公开仓库。

## 使用规则

- 每次写作前，如果用户提供了 `写作复盘.md`，先读取它，再生成大纲或正文。
- 复盘只记录可复用的问题，不记录一次性抱怨。
- 每条问题都要写成“下次如何避免”，不能只写“写得不好”。
- 如果某条复盘只适用于特定题材、平台、客户或文章类型，要标明适用范围。
- 如果复盘与当次风格素材冲突，优先尊重用户当次风格素材，并在大纲中提示冲突。
