# 输出质量 Checklist

Skill 输出报告后，按此清单检查是否能用。

## 内容质量
- [ ] 话题做了聚类，不是8条帖原样罗列
- [ ] 话题按讨论量/热度排了序
- [ ] 痛点按"频率 + 情绪强度"排序，最痛的在前
- [ ] 痛点用用户口吻描述，不是学术化措辞

## 选题可用性（最重要）
- [ ] 每个内容机会都有具体标题（像真的视频/文章标题）
- [ ] 每个选题标注了适合的下游 Skill（YouTube/Blog/FB/INS）
- [ ] 选题具体到能直接拿去做，不是"可以做散热相关内容"这种空话
- [ ] 选题回应了前面列出的痛点（有对应关系）

## 合规安全（不可漏）
- [ ] DPF delete / EGR delete 等话题标注了合规风险
- [ ] 涉及破解/绕过安全系统的话题，选题方向是诊断/维护，不是教删除
- [ ] 涉及维修操作的选题，提示了需技术验证

## 衔接下游
- [ ] 选题能直接复制给 fp_youtube_script / fp_blog_to_social
- [ ] 有没有能和已发内容形成系列的选题（加分项）

---

## 数据质量提醒
- 输入数据越全，聚类越准。建议至少 20-30 条热帖再跑分析
- 如果抓的帖子评论少，痛点提炼会偏弱，可补抓评论
- 不同 subreddit 侧重不同（r/heavyequipment 偏重型、r/lawncare 偏小型），
  跨板块数据一起喂，话题覆盖更全
