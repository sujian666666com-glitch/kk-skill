# 角色报告输出契约

所有角色（除 Independent Auditor 外）在 P3 阶段必须按此 schema 输出。
Independent Auditor 在 P4/P10 使用独立的 verdict 格式（见 [decision-memo.md](./decision-memo.md)）。

## 必填字段（7 项）

```markdown
## [角色名] 报告

### 观点
（对该项目的核心判断，2-3 句）

### 最大亮点
（项目最突出的优点，附证据）

### 最大问题
（最严重的问题，附证据或标注【推测】）

### 最大风险
（最可能导致失败的风险，说明触发条件）

### 最容易失败的位置
（具体到模块/流程/决策点，不泛泛而谈）

### 如果我是负责人，第一件做什么
（可执行的单一行动，不是建议清单）

### 评分：X/10
（必须解释评分理由，引用 [rubrics/scoring.md](../rubrics/scoring.md)）
```

## 证据标注规则

每条论断必须标注：
- 【证据】— 有明确来源（代码/文档/数据/公开信息）
- 【推测】— 基于经验推断，无直接证据
- 【假设】— 前提条件，未验证
- 【需要验证】— 可验证但当前未验证

`evidence_level: strict` 模式下，未标注视为无效结论。

## 反模式（角色必须避免）

- 只说优点不说问题（失职）
- 只说问题不给评分（逃避）
- 评分无解释（gaming）
- 复述其他角色观点（丧失独立性）
- 给出多条"第一件事"（违反单一行动原则）
