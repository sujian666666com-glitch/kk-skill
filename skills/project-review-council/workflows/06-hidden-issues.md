# Phase 6 — 隐藏问题挖掘

## 输入

P1-P5 全部产出。

## 处理

跳出已有视角，主动寻找盲区：
1. 哪些问题没有任何角色提到？
2. 最大的盲区是什么？
3. 哪些地方属于过度设计（YAGNI 违规）？
4. 哪些地方属于设计不足？
5. 哪些需求没有验证？
6. 哪些结论没有证据？

## 输出：盲区清单

```markdown
## 隐藏问题

### 没人提到的问题
- [问题] + [为什么没人提]

### 最大盲区
- [盲区] + [潜在后果]

### 过度设计（YAGNI 违规）
- [设计] + [为什么是过度的]

### 设计不足
- [缺失] + [为什么需要]

### 未验证的需求
- [假设的需求] + [如何验证]

### 无证据的结论
- [结论] + [需要什么证据]
```

## Exit Gate

- 6 类盲区每类至少 1 条（或明确声明"无"）
- 每条标注影响

## Decision Gate

| 情况 | 动作 |
|------|------|
| 发现新的 Fatal/High 风险 | 回退 P5 补入风险表 |
| 达标 | 进入 P7 |

## Retry / Rollback

- **Retry**：盲区分析表面化 → 追问"还有呢"至少 3 轮
- **Rollback**：发现遗漏严重 → 回退 P5
