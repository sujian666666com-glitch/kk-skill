# Phase 8 — 执行评估

## 输入

P5 风险分级表 + P6 隐藏问题 + P7 商业评估。

## 处理

输出最应该先做的 10 件事，必须排序。
每件事标注：
- **ROI** — 投入产出比（高/中/低）
- **风险** — 不做的后果
- **工作量** — S/M/L

## 输出：Top 10 行动项

```markdown
## 最应该先做的 10 件事（按优先级排序）

| 序 | 行动项 | ROI | 不做风险 | 工作量 |
|----|--------|-----|---------|--------|
| 1 | 修复支付绕过漏洞 | 高 | 资金损失(Fatal) | S |
| 2 | 建立数据库容灾 | 高 | 全站宕机(High) | M |
| 3 | 验证 D7 留存数据 | 高 | 增长盲区(Medium) | S |
| ... | | | | |
```

## Exit Gate

- 10 项齐全且排序
- 每项 3 字段齐全
- Fatal/High 风险对应的行动项排在前面

## Decision Gate

| 情况 | 动作 |
|------|------|
| 行动项未覆盖 Fatal 风险 | 驳回重排 |
| 排序与风险等级矛盾 | 修正 |
| 达标 | 进入 P9 |

## Retry / Rollback

- **Retry**：行动项不可执行 → 细化
- **Rollback**：遗漏 P5/P6 问题 → 回退补全
