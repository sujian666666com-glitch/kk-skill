# Phase 2 — 证据收集

> **模式**：为 P3-P9 提供证据基础。无证据的结论在后续阶段无效。

## 输入

P1 的项目画像 9 要素 + 原始材料。

## 处理

1. 遍历画像中的每个关键判断
2. 为每个判断标注证据等级
3. 对【需要验证】项，设计验证方案
4. 对【推测】项，标注推测依据

## 证据等级

| 标签 | 含义 |
|------|------|
| 【证据】 | 有明确来源（代码行 / 文档 / 数据 / 公开信息） |
| 【推测】 | 基于经验推断，无直接证据 |
| 【假设】 | 前提条件，未验证 |
| 【需要验证】 | 可验证但当前未验证 |

`evidence_level: strict` 模式下，所有结论必须标注。

## 输出：证据清单

```markdown
## 证据清单

| 判断 | 等级 | 来源/依据 | 验证方案（如需） |
|------|------|----------|-----------------|
| 用户增长 30%/月 | 【证据】 | GAIA 数据 2025-Q4 | — |
| 单位经济模型成立 | 【推测】 | 基于 LTV/CAC 估算 | 跑 3 个月真实数据 |
| 市场规模 100 亿 | 【需要验证】 | 第三方报告口径不一 | 委托独立调研 |
```

## Exit Gate

- 所有关键判断已标注证据等级
- 证据比例 ≥ `evidence_ratio_min`（默认 50%）

## Decision Gate

| 情况 | 动作 |
|------|------|
| 证据比例 < 50% | 回退：提示用户补充材料后重跑 P1-P2 |
| 关键假设未标注 | 驳回，补全后重新提交 |
| 达标 | 进入 P3 |

## Retry / Rollback

- **Retry**：证据不足 → 列出缺口让用户补充 → 重跑 P2
- **Rollback**：发现画像错误 → 回退 P1
