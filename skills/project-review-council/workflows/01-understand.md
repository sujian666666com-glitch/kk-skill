# Phase 1 — 项目理解

> **模式**：Pipeline 第 1 阶段，强制首先执行。只理解，不评价。

## 输入

用户提供的项目材料（描述 / 链接 / 代码仓库 / 文档 / 截图）。

## 处理

1. 通读全部材料
2. 提取项目画像 9 要素
3. 不做任何评价性判断
4. 信息不全时，明确列出未知项

## 输出：项目画像 9 要素

```markdown
## 项目画像

1. **项目目标** — 一句话：要做什么
2. **解决什么问题** — 谁、什么痛点、多大
3. **目标用户** — 精确画像（不是"所有人"）
4. **商业模式** — 怎么赚钱 / 谁付钱
5. **核心价值** — 用户为什么选你而不是替代品
6. **技术方案** — 技术栈、架构、关键依赖
7. **当前阶段** — 想法 / MVP / PMF / 扩张 / 成熟
8. **关键假设** — 哪些必须成立项目才成立
9. **关键风险** — 哪些会让项目失败

## 未知信息（如有）
- [列出材料中缺失的关键信息]
```

## Exit Gate

- 9 要素全部填写，或
- 缺失项已列入"未知信息"

## Decision Gate

| 情况 | 动作 |
|------|------|
| 未知项 >3 个且影响核心判断 | 向用户追问，不进入 P2 |
| 9 要素齐全 | 进入 P2 |
| 材料严重不足 | 提示用户补充后重跑 P1 |

## Retry / Rollback

- **Retry**：信息不全 → 列出未知项让用户补充 → 重跑 P1
- **Rollback**：后续阶段发现根本性误解 → 回退 P1 重新画像

## 用户检查点

输出画像后，向用户确认："这份理解准确吗？" 确认后再继续。
