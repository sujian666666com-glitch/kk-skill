# Phase 7 — 商业评估

## 输入

P1-P6 全部产出 + [config/weights.yaml](../config/weights.yaml) 的 11 维度。

## 处理

对 11 个维度逐一评分（0~10），每项必须附评分理由。
评分依据 [rubrics/scoring.md](../rubrics/scoring.md) 的判定锚点。
按 `weights.yaml` 的权重计算加权综合分。

## 输出：商业评估评分卡

```markdown
## 商业评估

| 维度 | 评分 | 权重 | 理由 |
|------|------|------|------|
| 商业价值 | 7/10 | 0.15 | 有清晰收入路径但未验证【证据】 |
| 技术价值 | 8/10 | 0.12 | 架构合理，可维护【证据】 |
| 用户价值 | 9/10 | 0.15 | 解决高频痛点，NPS 62【证据】 |
| 市场价值 | 6/10 | 0.10 | 市场大但竞争激烈【推测】 |
| 创新程度 | 5/10 | 0.08 | 微创新，非突破【证据】 |
| 竞争壁垒 | 4/10 | 0.15 | 壁垒薄弱，易被复制【需要验证】 |
| 可复制性 | 6/10 | 0.05 | 模式可复制（反向计分） |
| 商业模式 | 7/10 | 0.05 | 闭环但单位经济待优化 |
| 盈利能力 | 5/10 | 0.05 | 当前亏损，路径不清【推测】 |
| 增长潜力 | 7/10 | 0.10 | 有飞轮雏形 |
| 全球化潜力 | 4/10 | 0.05 | 本地化依赖强【需要验证】 |

**加权综合分**：X.X/10
```

## Exit Gate

- 11 维度全部评分
- 每项理由标注证据等级
- 加权综合分计算正确

## Decision Gate

| 情况 | 动作 |
|------|------|
| 评分理由无证据标注 | 驳回补全 |
| 加权分计算错误 | 修正 |
| 达标 | 进入 P8 |

## Retry / Rollback

- **Retry**：某维度评分无理由 → 重跑该维度
- **Rollback**：评估依赖错误事实 → 回退 P2 补证据
