# 决策矩阵

## 四档建议

| 建议 | 触发条件 | 含义 |
|------|---------|------|
| **继续** | 无 Fatal + High ≤2 且有路径 + 商业评估均分 ≥6 | 全速推进 |
| **继续（加速）** | 无 Fatal + 商业评估均分 ≥7 + 壁垒评分 ≥6 | 加大投入 |
| **暂停** | 存在 Fatal 或 ≥3 项 High 无解法 | 停下解决关键风险后再评估 |
| **重构** | ≥3 项 High 且根因为架构/模式 | 推倒重做部分系统 |
| **放弃** | Fatal 不可解 或 商业评估均分 <4 | 止损退出 |

## 决策流程

```
P5 有 Fatal 风险？
├─ 是 → Fatal 可解？
│       ├─ 是 → 解决成本可接受？→ 继续（带 P0）
│       └─ 否 → 放弃
└─ 否 → High 风险 ≥3？
        ├─ 是 → 根因是架构/模式？→ 重构
        └─ 否 → High ≤2？
                ├─ 是 → 均分 ≥7？→ 继续（加速）/ 继续
                └─ 否 → 均分 <4？→ 放弃 / 暂停
```

## 决策修正（P10 红蓝对抗后）

- 若 Red Team 成立论据 > Blue Team 成立论据的 1.5 倍 → 决议降级（继续→暂停）
- 若 Auditor 发现 P9 致命疏漏 → 强制回退 P9 重做决议

## "如果这是你的项目"约束

P9 必须回答"你会不会继续做"。回答必须基于：
1. 上述矩阵的客观结果
2. 个人风险偏好声明（明确说明偏好激进/保守）
3. 不能只说"会"或"不会"，必须给理由
