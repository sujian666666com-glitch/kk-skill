# 风险等级标准

## 四级分类

| 等级 | 定义 | 影响 | 处置 |
|------|------|------|------|
| **Fatal** | 致命风险，项目可能不可用 / 违法 / 资金断裂 | 生存级 | P0 立即阻断，解决前不推进 |
| **High** | 核心价值受损 | 业务级 | P1 本阶段必修 |
| **Medium** | 局部退化或可控风险 | 功能级 | P2 下阶段修 |
| **Low** | 体验问题或优化项 | 体验级 | P3 排期 |

## 评估维度（每项必填）

| 字段 | 说明 |
|------|------|
| **影响范围** | 受影响的用户/功能/业务（具体，不泛指） |
| **发生概率** | 高(>50%) / 中(20-50%) / 低(<20%) / 已发生 |
| **解决成本** | S(<1人日) / M(1-5人日) / L(>5人日) |
| **优先级** | P0/P1/P2/P3，由等级×概率×成本推导 |

## 优先级推导矩阵

| 等级 \ 概率 | 高 | 中 | 低 |
|------------|----|----|-----|
| Fatal | P0 | P0 | P1 |
| High | P1 | P1 | P2 |
| Medium | P2 | P2 | P3 |
| Low | P3 | P3 | P3 |

## Fatal 风险示例

- 安全漏洞导致数据泄露 / 资金损失
- 商业模式无法盈利且无改善路径
- 核心假设被证伪（伪需求）
- 合规问题导致业务非法
- 单点故障无恢复方案

## 特殊规则

- **Fatal 即使概率低也至少 P1**（黑天鹅防护）
- **多个 Medium 叠加可升级为 High**（需在 P5 显式说明）
- **未验证项默认按"中概率"处理**（保守原则）
