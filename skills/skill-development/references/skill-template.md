# SKILL.md 标准模板

## 模板 A：处理型 Skill

适用于：输入 → 处理 → 输出 的线性任务

```markdown
---
name: {技能名}
version: 1.0.0
description: {英文描述，包含 WHAT 和 WHEN}
description_zh: {中文描述}
user-invocable: true
argument-hint: {输入提示}
---

# {技能标题}

你是一位 {角色}，专门负责 {核心职责}。

## 输入识别

根据用户输入的类型，走不同的处理路径：

- **类型A**（{特征}）→ {处理路径}
- **类型B**（{特征}）→ {处理路径}
- **信息不足** → 追问 {缺失信息}

## 处理流程

### 第一步：{步骤名}
{具体指令}

### 第二步：{步骤名}
{具体指令}

### 第三步：{步骤名}
{具体指令}

## 输出格式

{输出结构定义}

## 质量检查

- [ ] {检查项1}
- [ ] {检查项2}

## If Connectors Available

If **{品类}** is connected:
- {增强能力描述}

If no connectors available:
- 输出为本地文件（默认行为）
```

## 模板 B：分析型 Skill

适用于：数据分析、评估、诊断类任务

```markdown
---
name: {技能名}
version: 1.0.0
description: {英文描述}
description_zh: {中文描述}
user-invocable: true
argument-hint: {输入提示}
---

# {技能标题}

你是一位 {角色}。你的任务是 {核心职责}。

## 输入识别

- **有数据**（上传了文件/粘贴了数据）→ 走分析流程
- **无数据**（只有问题描述）→ 先收集数据，再分析
- **有历史报告**（之前的分析结果）→ 走对比分析流程

## 分析框架

### 维度1：{维度名}
- 指标：{具体指标}
- 基准：{参考标准}
- 判断规则：{如何判断好坏}

### 维度2：{维度名}
...

## 输出格式

### 评分总览
| 维度 | 评分 | 说明 |
|------|------|------|

### 详细分析
{每个维度的详细分析}

### 改进建议
{按优先级排列的行动项}

## If Connectors Available

If **{品类}** is connected:
- {增强能力描述}

If no connectors available:
- 输出为本地文件（默认行为）
```

## 模板 C：知识库型 Skill

适用于：纯知识存储，被其他 Skill 引用，用户不直接调用

```markdown
---
name: {技能名}
version: 1.0.0
description: {英文描述}
description_zh: {中文描述}
user-invocable: false
---

# {知识库标题}

本知识库包含 {领域} 的核心知识，供其他 Skill 引用。

## 使用方式

其他 Skill 通过以下方式引用：

参考 [{知识主题}](references/{文件名}.md) 获取 {具体内容}。

## 知识索引

| 文件 | 内容 | 使用场景 |
|------|------|---------|
| {file1.md} | {内容概要} | {什么时候用} |
| {file2.md} | {内容概要} | {什么时候用} |

## 更新日志

| 版本 | 日期 | 变更内容 |
|------|------|---------|
| 1.0.0 | {日期} | 初始版本 |
```
