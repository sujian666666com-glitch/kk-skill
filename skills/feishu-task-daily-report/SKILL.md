# Feishu 任务清单 → 日报 → 备忘录

按以下固定流程执行：

1. 解析任务表链接
- 用户提供 `/base/` 或 `/wiki/` 链接时，先解析 app_token 与 table_id。
- 若链接带的 table 参数无效，回退到表列表并选择“任务清单”表。

2. 拉取任务记录
- 读取字段至少包含：任务标题、创建时间、创建人、负责人、关注人、任务完成状态。
- 需要分页拉全量记录。

3. 筛选“今日且与我相关”（含回退）
- 先查今天：当天 00:00:00~23:59:59（Asia/Shanghai）。
- 若今天结果为空，自动回退查昨天（同样时间范围）。
- 与我相关满足其一：
  - 创建人是用户；
  - 负责人包含用户；
  - 关注人包含用户。

4. 提炼日报条目
- 只保留可汇报推进项。
- 合并重复表述，输出 3~6 条。
- 不输出任务ID、链接、负责人等冗余字段。

5. 固定输出格式
严格使用：

今日工作总结：
1.xxx
2.xxx
3.xxx

6. 直接发送给用户
- 生成后直接把日报正文发送给用户。
- 不写入备忘录。

7. 交付
- 发送时保持固定格式，不附加任务ID、链接等冗余信息。

## 用户偏好（强约束）
- 日报必须简洁。
- 必须用“今日工作总结：”开头。
- 必须编号分条。
- 不要任务ID等技术字段。
